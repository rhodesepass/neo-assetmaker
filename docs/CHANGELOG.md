# 更新日志

### v2.1.0

**视频系统重写**
- 视频解码引擎从 OpenCV 切换到 PyAV，新增 OpenGL 硬件渲染器
- 帧预取缓冲区 + PBO (Pixel Buffer Object) 异步上传，减少 GPU 等待
- 视频预览后台线程化：帧读取和处理移至 `FrameReaderThread(QThread)`，长视频加载和播放不再阻塞 UI
- 内置丢帧策略，确保 30fps 视频流畅播放
- Tab 切换无延迟，预览页面不再互相卡住
- 修复视频预览拉伸变形：`_update_mvp()` 添加所有旋转角度的 letterbox 宽高比校正
- 修复裁剪框只能水平移动：初始大小改为最大尺寸的 75%，四周留出自由移动空间
- PyOpenGL 打包支持和运行时防护

**新增任意角度旋转**
- 支持 0-359° 任意整数角度旋转（不限于 0°/90°/180°/270°）
- 时间轴新增旋转 SpinBox 控件和 ↺/↻ 步进按钮
- 正交角度使用 `cv2.rotate` 快速路径（约 10 倍于 warpAffine）

**新增拖放文件导入**
- 支持将视频/图片文件直接拖放到预览区域导入
- 动态上下文感知（根据当前标签页接受对应文件类型）
- 半透明覆盖层 + 虚线边框视觉反馈

**新增崩溃恢复系统**
- 异常退出后自动检测未保存的临时文件
- 崩溃恢复对话框，支持选择恢复项目

**新增自动保存功能**
- 定期自动保存项目配置
- 可配置保存间隔，备份文件自动管理

**新增 SSH 远程上传管理**
- 侧边栏新增远程管理入口（WiFi 图标）
- SSH 连接配置：地址、端口、用户名、密码、远程路径
- 一键上传文件到通行证设备，实时进度显示
- 上传完毕后可选自动重启通行证程序
- 修复使用自动上传后 drmapp 无法启动外部应用的问题

**新增素材论坛扩展模块**
- 用户认证系统
  - OAuth2 + PKCE 登录流程
  - FIDO2/WebAuthn 安全密钥认证
  - Keyring 集成的令牌安全存储
  - 登录/注册/登出/令牌自动刷新
- 素材市场
  - 素材浏览（网格/列表视图）
  - 搜索和过滤面板
  - 收藏和下载功能
  - 本地素材库管理
- 下载管理系统
  - 并发下载引擎（QThreadPool + QRunnable）
  - 暂停/恢复/取消/重试
  - 实时进度、速度和 ETA 显示
- USB 设备管理
  - USB 设备自动检测和轮询监控
  - MTP (Media Transfer Protocol) 实现
  - 文件浏览和传输功能
  - 设备信息卡片（制造商/产品/存储容量）
- 统一服务管理器（延迟初始化、优雅关闭）

**UI 改进**
- 主题图标、间距和配色优化
- 窗口圆角真正裁剪（`WA_TranslucentBackground` + `paintEvent`），四角不再漏色
- 补齐所有四角的 `border-radius`（左上、左下之前缺失）
- 背景图片改由 `paintEvent` 绘制，支持圆角裁剪
- 导出时支持自定义 FFmpeg 路径

**编译后闪退修复**
- QApplication 之前设置 `AA_ShareOpenGLContexts`，解决 QOpenGLWidget + QWebEngineView COM 冲突
- `check_dependencies()` 移到 QApplication 之后，避免 QtWebEngine COM 初始化冲突 (Windows fatal exception 0x8001010d)
- stderr/stdout 重定向到文件而非 StringIO，崩溃诊断信息不再丢失
- 全局 `sys.excepthook` 安全网，防止 PyQt6 slot 未处理异常导致 abort
- 冻结环境下 12 处 `__file__` 路径修复（使用 `get_app_dir()`）

**构建系统**
- 新增可复用的 GitHub Actions 构建工作流
- uv 升级到 0.10.10，添加 lock 检查
- CI 修复：uv setup、PyOpenGL fallback、额外依赖安装

**Bug 修复**
- 修复 `QDialog` 未导入导致崩溃恢复对话框功能失效
- 修复 `paintGL()` 缺少异常保护可能导致程序崩溃
- 修复固件下载函数在 `./bin` 不存在时崩溃
- 修复操作下拉框异常截断（Qt `\t` 宽度计算错误，改用 shortcut）

**产品重命名**
- 产品名称从"明日方舟通行证素材制作器"更名为"明日方舟通行证素材工具箱"

**Qt 框架迁移**
- 移除 qtpy 抽象层依赖，全模块直接使用 PyQt6 导入

**素材论坛模块重构**
- 模块品牌统一：素材商城 → 素材论坛（MaterialMarketWidget → MaterialForumWidget）
- 页面懒加载：所有页面延迟到首次显示时初始化，减少启动时服务创建开销
- 设置页面新增登录按钮，未登录时可直接跳转登录
- 配置目录创建失败时自动回退到系统临时目录

**安装器增强**
- AppMutex 互斥量保护：安装/升级时自动检测并提示关闭正在运行的应用
- MinVersion=10.0：在不支持的 Windows 版本上安装时友好提示
- CloseApplications：自动检测占用安装目录文件的进程
- 版本号动态注入：本地构建自动从 pyproject.toml 同步版本号到安装脚本

**视频系统优化**
- 浮点精度修复：帧索引计算从 `int()` 改为 `round()`，消除累积误差
- 帧缓冲区同步：`threading.Condition` 替代自旋轮询，显著降低 CPU 占用
- 自适应预读策略调优：缓冲区上限扩展到 16 帧，检查频率提升至 0.5 秒
- QTimer 精准模式（`PreciseTimer`），提升播放帧率准度
- 导出完成后验证实际写入帧数，偏差 >10% 时发出警告

**固件烧录增强**
- 细粒度进度追踪信号（驱动检查 → 文件准备 → 设备连接 → xfel → DFU 各阶段）
- 新增 DFU 设备检测和等待逻辑
- 烧录流程分阶段：xfel spinand → DFU boot 分区 → DFU rootfs 分区

**SSH 远程管理改进**
- 新增独立 `SshRestartWorker` 线程类，替代临时 `threading.Thread`
- 上传和重启过程通过信号实时反馈日志
- 素材编辑流程优化：统一使用 `_find_epconfig()` 查找配置文件，支持扁平/递归两种目录结构
- 日志输出自动限制 1000 行，避免内存膨胀

**Bug 修复（增量）**
- 修复设置页面 SSH IP 地址配置键名错误（`'z'` → `'ssh_ip_address'`）
- 恢复被注释掉的界面缩放设置功能
- 修复日志系统退出时处理器未正确关闭导致的资源泄漏
- 修复应用退出时日志缓冲区未刷新（添加 `logging.shutdown()`）
